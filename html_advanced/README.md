# Advanced HTML

A practical guide to **advanced HTML concepts**, best practices, and modern techniques for building accessible, performant, and maintainable web pages.

---

## Overview

Advanced HTML focuses on writing **meaningful, accessible, and efficient markup** using modern standards. It emphasizes semantics, accessibility, native browser features, and clean structure.

---

## Semantic HTML

Semantic elements describe the purpose of content clearly.

Examples include:

- `<header>`, `<nav>`, `<main>`
- `<section>`, `<article>`, `<aside>`
- `<footer>`

Benefits:

- Better accessibility
- Improved SEO
- Easier maintenance

---

## Accessibility (A11y)

Accessibility ensures websites work for everyone.

Key practices:

- Use semantic HTML first
- Always provide `alt` text for images
- Ensure keyboard navigation
- Use ARIA only when necessary

---

## Forms & Media

### Forms

HTML provides native validation:

- `required`, `pattern`, `min`, `max`
- Input types like `email` and `url`

Use native features before JavaScript.

### Media

- Use responsive images (`srcset`, `<picture>`)
- Include captions for video and audio
- Prefer SVG for icons and logos

---

## Performance, SEO & Security

### Performance

- Lazy load images
- Reduce unnecessary elements
- Use modern image formats

### SEO

- Use proper headings
- Write clear `<title>` and meta descriptions
- Declare page language

### Security

- Escape user input
- Avoid inline scripts
- Use safe link attributes

---

## Best Practices

- Keep HTML clean and semantic
- Prefer HTML solutions over JavaScript
- Test with accessibility tools
